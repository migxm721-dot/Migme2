# Contributing Guidelines

Thank you for contributing to migme!

---

## Getting started

1. Fork the repository on GitHub.
2. Follow the [SETUP_GUIDE.md](../SETUP_GUIDE.md) to set up your local development environment.
3. Create a feature branch: `git checkout -b feature/my-feature`
4. Make your changes.
5. Run tests: `./gradlew test`
6. Build a debug APK to verify: `./gradlew assembleDebug`
7. Commit and push your branch.
8. Open a Pull Request against `main`.

---

## Code style

- Follow existing code conventions in the file you are editing.
- Use meaningful variable and method names.
- Keep methods small and focused.

---

## Pull request checklist

- [ ] Code compiles without warnings
- [ ] Debug APK builds successfully
- [ ] Unit tests pass
- [ ] New functionality is documented
- [ ] No secrets or credentials are included
